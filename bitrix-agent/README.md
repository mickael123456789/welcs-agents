# 🕵️ Ежедневный надзорный агент по Bitrix24

Каждый день агент заходит в Bitrix через REST API, читает диалоги с клиентами,
внутренние чаты, сделки/лиды и задачи, прогоняет всё через Claude и присылает
вам на почту короткую сводку:

- 🔴 **Срочно / требует вмешательства** — где руководителю надо вмешаться сейчас
- ⚠️ **Сложности и узкие места** — системные проблемы
- 👏 **Похвалить** — кого и за что
- 👎 **Поругать / обратить внимание** — несвоевременные/неверные/отсутствующие ответы
- 📊 **Пульс дня** + **Индекс внимания N/10**

---

## Архитектура

```
run.py
  ├─ collect.py        ← тянет данные из Bitrix (открытые линии, чаты, CRM, задачи)
  │    └─ bitrix_client.py
  ├─ analyze.py        ← Claude формирует сводку (с кэшированием системного промпта)
  └─ notify.py         ← письмо на почту (SMTP)
```

## Установка (один раз)

### 1. Зависимости

```bash
cd bitrix-agent
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
```

### 2. Создать входящий вебхук в Bitrix24

1. В Bitrix: **Разработчикам** → **Другое** → **Входящий вебхук**
   (или `Настройки → Разработчикам`).
2. Выдать права (scope): **`crm`**, **`im`**, **`imopenlines`**, **`tasks`**, **`user`**.
3. Скопировать URL вида
   `https://ВАШ-ПОРТАЛ.bitrix24.ru/rest/1/ABCDEF1234567890/`

> ⚠️ Вебхук работает от имени создавшего его пользователя и видит **только те
> чаты и открытые линии, куда у этого пользователя есть доступ**. Чтобы агент
> видел клиентские диалоги — создавайте вебхук под аккаунтом, который состоит в
> очередях открытых линий (или под админом с доступом ко всем линиям).

### 3. Пароль приложения Gmail

Обычный пароль Gmail для SMTP не подойдёт. Нужен **App Password**:
`myaccount.google.com` → Безопасность → (включить 2FA) → **Пароли приложений**.
Получите 16-символьный пароль.

### 4. Заполнить `.env`

```bash
cp .env.example .env
# открыть .env и вставить: BITRIX_WEBHOOK_URL, ANTHROPIC_API_KEY,
# SMTP_USER, SMTP_PASSWORD
```

## Проверка

```bash
# 1. Просто собрать данные и посмотреть, что видит агент (без Claude и письма):
python run.py --save-raw            # → reports/raw-YYYY-MM-DD.json

# 2. Полный анализ без отправки письма (печать в консоль):
python run.py --dry

# 3. Боевой прогон с письмом:
python run.py
```

Если открытые линии/чаты приходят пустыми — проверьте права вебхука и членство
пользователя в очередях (см. предупреждение выше).

## Режимы: утро и вечер

- `python run.py --mode morning` — «что было за ночь», окно 14ч, акцент на заезды/уборки сегодня.
- `python run.py --mode evening` — «итоги дня», окно 12ч, акцент на скорость операторов и что осталось на завтра.

## Ежедневный запуск — GitHub Actions (рекомендуется)

Бесплатный облачный cron, работает при выключенном Mac. Полная инструкция —
в **[DEPLOY.md](DEPLOY.md)**: создать приватный репо → добавить 4 секрета →
готово. Запуск автоматически в 08:00 (morning) и 20:00 (evening).

Альтернатива — обычный cron на всегда включённой машине (VPS):

```cron
0 8 * * *  cd /путь/bitrix-agent && /путь/.venv/bin/python run.py --mode morning >> reports/cron.log 2>&1
0 20 * * * cd /путь/bitrix-agent && /путь/.venv/bin/python run.py --mode evening >> reports/cron.log 2>&1
```

## Настройка под вашу компанию

- **Что считать срочным / кого хвалить-ругать** — текст роли в `analyze.py`,
  переменная `SYSTEM_PROMPT`. Меняйте под свои критерии и тон.
- **Глубина выгрузки** — `LOOKBACK_HOURS`, `MAX_DIALOGS`, `MSGS_PER_DIALOG` в `.env`.
- **Получатели** — `REPORT_TO` (можно несколько через запятую).

## Безопасность

`.env` и `reports/` уже в `.gitignore`. Вебхук = полный доступ к данным портала —
храните его как пароль. Если утёк — отзовите вебхук в Bitrix и создайте новый.
